# 贡献指南

欢迎从目录维护、内容写作、资源补充、模板优化等方面参与共建。本页重点回答两个问题：

- 如何写一篇符合本站规范的文章
- 如何把文章接入索引（目录页与侧边栏）

## 你可以贡献什么

- 新增学习文章：导论、方法、专题综述、案例拆解
- 维护目录页：补充某学科分支下的资源条目
- 修订已有内容：纠错、更新失效链接、补充版本信息
- 优化工具页：沉淀模板、检索策略、写作规范

## 写文章前先选位置

先确定文章属于哪个栏目，再创建文件，避免后续大改路径：

- 使用指南：`docs/guide/`
- 资源目录：`docs/catalog/`
- 工具与模板：`docs/tools/`
- 共建与社区：`docs/community/`
- 关于项目：`docs/about/`

命名建议使用短横线英文文件名，例如：`case-method.md`、`evidence-review.md`。

## 如何写文章

### 1. 复制最小骨架

```md
# 标题（清晰描述主题）

## 这篇文章解决什么问题

一句话说明适用场景与目标读者。

## 核心内容

- 要点 1
- 要点 2
- 要点 3

## 参考与来源

- 资料名 / 链接 / 版本或年份
```

### 2. 写作要求

- 标题直接表达问题，不用口号式标题
- 每篇至少说明：适用人群、使用场景、前置知识
- 引用资源时注明来源、版本/年份、获取路径
- 涉及法条或案例时，尽量给出准确条文名或案号
- 使用二级、三级标题组织结构，避免大段无层级文字

### 3. 内容质量底线

- 可验证：结论能追溯到来源
- 可执行：读者看完能按步骤操作
- 可维护：不要写“仅当下有效”的表述而无时间说明

## 如何更新索引（重点）

新文章写完后，至少要更新 2 处索引，否则用户很难找到。

### 1. 更新栏目内目录页

以资源目录为例，你新增了 `docs/catalog/basic.md` 相关内容，就应在 `docs/catalog/overview.md` 增补对应条目或说明，让“总览页”能看见新增内容。

### 2. 更新站点侧边栏

编辑 `docs/.vitepress/config.mts` 的 `themeConfig.sidebar`，把新页面加到对应分组。

示例（以 `/guide/` 为例）：

```ts
{
  text: "使用指南",
  items: [
    { text: "平台介绍", link: "/guide/intro" },
    { text: "学习路径", link: "/guide/roadmap" },
    { text: "你的新文章标题", link: "/guide/your-new-page" }
  ]
}
```

注意：

- `link` 不带 `.md` 后缀
- 路径必须与文件位置一致
- 文案要和页面标题一致，降低检索成本

### 3. 可选：更新导航或交叉链接

若是一级入口内容，可考虑同步更新：

- 顶部导航：`docs/.vitepress/config.mts` 的 `nav`
- 相关页面中的“延伸阅读”或“下一步”链接

## 提交前检查清单

- 新文件已放在正确目录，命名规范
- 页面标题、摘要、小标题完整
- 来源信息完整（链接 + 版本/年份）
- 已更新对应索引（`overview` 或 `sidebar`）
- 本地预览无报错，链接可点击

## 推荐提交流程

1. 先阅读 `docs/community/submit.md` 的提交清单。
2. 小步提交：文章内容与索引更新尽量同一个 PR。
3. 在 PR 描述中写明：改动文件、目标读者、主要增量。
4. 如有争议内容，在 PR 中附参考依据与取舍理由。
